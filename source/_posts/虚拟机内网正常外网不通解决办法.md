# 问题现象
重启esxi后，ubuntu server 16.04虚拟机网络不通，ping路由器ip通的，但是ping百度不通。

# 问题定位
内网通，ip a查看网卡启动正常，ping内网也是通的，说明是dns解析有问题，查看/etc/resolv.conf，配置为空。

# 解决方案
在/etc/resolv.conf 增加nameserver 192.168.2.1（路由器ip）。
